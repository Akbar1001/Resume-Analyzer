Check if .env is properly ignored:
  $ git check-ignore .env
  Output: .env ← Means it's properly ignored!

Verify no .env files in git:
  $ git status
  # .env should NOT appear in the list

Search for accidentally committed credentials:
  $ git log --all --full-history -- .env
  # Should not show existing .env files

Scan for hardcoded secrets:
  $ git grep -i "AIzaSy"
  # Should only find in .env.example

Q: Where's my .env file?
A: It's hidden on your local machine. Copy .env.example to .env

Q: Why can't I see .env in git status?
A: Good! That's the point. It's git-ignored for security.

Q: Can I share my .env with the team?
A: NO! Each person needs their own local .env. Share only .env.example

Q: What if I accidentally committed .env?
A: See SECURITY_GUIDE.md → Handling Compromised Credentials

Q: How do I deploy safely?
A: See DEPLOYMENT.md or FREE_DEPLOYMENT.md (use platform dashboard)

Q: I'm getting "API key not set" error
A: Make sure your .env file exists and has GOOGLE_GENAI_API_KEY
